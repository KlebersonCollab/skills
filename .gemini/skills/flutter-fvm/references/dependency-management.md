# Dependency Management: Flutter with FVM

Managing dependencies in Flutter with FVM is simple and follows the Dart ecosystem (Pub) standards, but always prefixed with `fvm`.

## 1. Basic Pub Commands via FVM

| Action | Command |
|------|---------|
| Install dependencies | `fvm flutter pub get` |
| Add package | `fvm flutter pub add <package>` |
| Remove package | `fvm flutter pub remove <package>` |
| Update packages | `fvm flutter pub upgrade` |
| Check vulnerabilities | `fvm flutter pub deps` |

## 2. Production vs. Development Dependencies

In `pubspec.yaml`:

```yaml
dependencies:
  flutter:
    sdk: flutter
  http: ^1.2.0
  riverpod: ^2.5.1

dev_dependencies:
  flutter_test:
    sdk: flutter
  flutter_lints: ^3.0.1
  mocktail: ^1.0.3
  build_runner: ^2.4.8 # For code generation
```

## 3. Semantic Versioning (SemVer)

- `^1.2.3`: Allows non-breaking updates (major = 1).
- `1.2.3`: Pins exactly to version 1.2.3.
- `any`: Any version (not recommended for production).

## 4. The `pubspec.lock` file

This file is automatically generated and contains the EXACT versions of each installed package.

**GOLDEN RULE**: Always add `pubspec.lock` to version control (Git). This ensures that the entire team and CI/CD use the same dependency versions.

## 5. Code Generation (`build_runner`)

Many popular packages (Freezed, JSON Serializable, Riverpod Generator) depend on code generation.

Always execute via FVM:

```bash
# Run once
fvm flutter pub run build_runner build --delete-conflicting-outputs

# Run in watch mode (active development)
fvm flutter pub run build_runner watch --delete-conflicting-outputs
```

## 6. Pub Cache

Pub maintains a global package cache. If you encounter strange dependency errors, try clearing the cache:

```bash
fvm flutter pub cache clean
fvm flutter pub get
```

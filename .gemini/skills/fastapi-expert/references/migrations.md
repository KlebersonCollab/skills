# Database Migrations with Alembic (Async)

The management of database schema changes must be versioned and automated.

## 1. Asynchronous Setup with UV

Always use the Alembic `async` template for compatibility with FastAPI.

```bash
uv add alembic sqlalchemy asyncpg  # or aiosqlite
uv run alembic init -t async migrations
```

## 2. Configuration of `env.py`

In the `migrations/env.py` file, integrate your Pydantic configurations and your SQLAlchemy models:

```python
from app.core.config import settings
from app.models.base import Base  # Metadata containing all models

# ...
config = context.config
config.set_main_option("sqlalchemy.url", settings.DATABASE_URL)
target_metadata = Base.metadata
# ...
```

## 3. Migration Workflow

| Command | Description |
|---------|-----------|
| `uv run alembic revision --autogenerate -m "desc"` | Creates a new migration detecting model changes. |
| `uv run alembic upgrade head` | Applies all pending migrations. |
| `uv run alembic downgrade -1` | Reverts the last applied migration. |
| `uv run alembic history` | Lists the migration history. |

## 4. Best Practices
- **Manual Review**: Always review the script generated by `--autogenerate` before applying. It can fail on column renames.
- **Atomic Migrations**: Each migration should be small and focus on a single logical change.
- **Production Safety**: NEVER run `upgrade head` automatically in production without a recent database backup.

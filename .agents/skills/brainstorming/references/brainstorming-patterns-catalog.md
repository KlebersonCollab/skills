# Brainstorming Patterns Catalog

This catalog provides structured techniques for facilitating high-quality brainstorming sessions, based on proven prompt engineering and design thinking patterns.

---

## 🛠️ Pattern Selection Guide

Choose the right technique based on your current objective:

| If you need... | USE this Pattern |
|----------------|------------------|
| Diverse viewpoints on the same problem | **Perspective Multiplication** |
| Ideas that are immediately implementable | **Constraint Variation** |
| Identifying risks and challenging assumptions | **Inversion (Worst Possible Idea)** |
| Breakthrough "out of the box" innovation | **Analogical Transfer** |
| Long-term strategic decisions | **Scenario/Time Mapping** |
| Refining an existing product or process | **SCAMPER Framework** |

---

## 1. Perspective Multiplication
*Generating ideas from multiple viewpoints and stakeholder angles.*

### 1A: Role-Based Persona
**Template:**
```
You are a [specific role with 2-3 defining characteristics]. 
Given [context/challenge], generate [number] [type of ideas]. 
For each idea, explain your reasoning from your unique perspective as [role].
```

### 1B: Stakeholder Perspective Grid
**Template:**
```
Generate [number] ideas for [challenge] considering the perspective of:
1. [Stakeholder 1] - key concern: [specific pain point]
2. [Stakeholder 2] - key concern: [specific pain point]
3. [Stakeholder 3] - key concern: [specific pain point]

Format as a table: Idea | Stakeholder 1 Impact | Stakeholder 2 Impact | Trade-offs.
```

---

## 2. Constraint Variation
*Systematically exploring the idea space through artificial limitations.*

### 2A: Progressive Constraint Tightening
**Template:**
```
Generate ideas for [challenge] under different constraint scenarios:
- Scenario A (Baseline): Unlimited budget, 6-month timeline.
- Scenario B (Moderate): 50% budget reduction, 3-month timeline.
- Scenario C (Extreme): 90% budget cut, 1-week timeline, use only current resources.

Identify: Which ideas appear in all? What innovations emerge only under extreme constraints?
```

---

## 3. Inversion
*Using reverse thinking to find novel solutions and avoid failure.*

### 3A: "Worst Possible Idea"
**Template:**
```
Instead of asking "How do we improve X?", let's ask the opposite: "How do we make X the worst possible?".
Generate 5-7 ideas to make [product/process] terrible. 
For EACH bad idea, reverse it to identify what we should protect or strengthen.
```

### 3B: "What Would Make This Fail?" (Pre-mortem)
**Template:**
```
Identify 10 ways this plan could fail completely.
For each failure mode: Probability | Severity | Mitigation Strategy.
```

---

## 4. Analogical Transfer
*Applying patterns and solutions from different domains to the current problem.*

### 4A: Cross-Domain Analogy
**Template:**
```
How would [Domain/Company X] solve our challenge?
Examples: 
- How would Netflix solve this? (Focus: Retention/Content)
- How would a luxury hotel solve this? (Focus: Anticipatory service)
- How would nature solve this? (Focus: Biological systems/efficiency)

Extract the core principle and adapt it to our context.
```

## 5. SCAMPER Framework
*Classic method for modifying existing ideas.*

- **S**ubstitute
- **C**ombine
- **A**dapt
- **M**odify
- **P**ut to another use
- **E**liminate
- **R**everse

---

## 6. Future Scenario Mapping (Time Travel)
**Template:**
```
Imagine [product/business] in the year [2030/2035].
1. What fundamental problem do we solve (or stop solving)?
2. What has become obsolete?
3. What new capabilities are standard?

Reverse-engineer: What 5 strategic initiatives do we need to take TODAY?
```
